## 2. ファイル構成と責務

```
Diffchecker/
├── Program.cs              … エントリポイント（Shift_JISエンコーディング登録含む）
├── MainForm.cs             … メイン画面（UI構築・イベント処理・エンコーディング検出）約400行
├── DiffLine.cs             … データモデル（DiffStatus列挙型 + DiffLineクラス）約40行
├── DiffEngine.cs           … LCSベース差分検出アルゴリズム 約180行
├── DiffRenderer.cs         … 色分け描画 + スクロール連動 約130行
├── DiffReporter.cs         … テキストレポート生成 約115行
└── Common/                 … DesktopKitシリーズ共通ライブラリ
    ├── BaseForm.cs         … 共通ウィンドウ設定（サイズ・フォント・ステータスバー）
    ├── AppSettings.cs      … JSON設定の読み書き
    ├── FileDialogHelper.cs … ファイル/フォルダダイアログのラッパー
    └── StatusHelper.cs     … ステータスバー表示ヘルパー（Info/Success/Error色分け）
```

**設計方針:**
- 役割ごとにクラスを分離: データ(DiffLine) → アルゴリズム(DiffEngine) → 表示(DiffRenderer) → 出力(DiffReporter)
- Commonライブラリは FolderViewer と共有の別プロジェクト
- デザイナーファイル（.Designer.cs）は使用せず、UIはすべてコードで構築

## 3. 主要クラスの役割

### MainForm（メイン画面）
- UIコントロールの生成・配置
- ファイル選択（ダイアログ + ドラッグ&ドロップ）
- 拡張子の一致チェック
- エンコーディング自動検出（UTF-8 BOM → UTF-8 → Shift_JIS）
- DiffEngine / DiffRenderer / DiffReporter の呼び出し
- ステータスバーの更新

### DiffLine（データモデル）
- 差分結果1行分のデータ構造
- ファイル1の行番号・内容、ファイル2の行番号・内容、差分種類（Equal/Added/Deleted/Modified）を保持

### DiffEngine（差分検出）
- LCS（最長共通部分列）のDPテーブル構築
- バックトラックによるEqual/Deleted/Addedの分類
- 後処理: 連続するDeleted→Addedの組をModifiedに変換

### DiffRenderer（色分け描画）
- DiffLineリストを左右のRichTextBoxに色分け表示
- 行番号フォーマット、プレースホルダ（`---`）の挿入
- Win32 APIによる左右スクロール連動

### DiffReporter（レポート出力）
- DiffLineリストからテキストレポートを生成
- 2セクション構成: 全行マーク付き表示 + 差分サマリー
- UTF-8 BOM付き・CRLF改行で出力

## 4. データの流れ

```
[ファイル1] [ファイル2]
    ↓          ↓        File.ReadAllLines + DetectEncoding
[string[]]  [string[]]
    ↓          ↓
    └────┬─────┘
         ↓
   [DiffEngine.Compare]
   LCSテーブル構築 → バックトラック → Modified変換
         ↓
   [List<DiffLine>]  ← 比較結果（全行分）
    ↓           ↓
    ↓      [DiffReporter.Export]
    ↓           ↓
    ↓      [.txtレポートファイル]
    ↓
   [DiffRenderer.Render]
    ↓           ↓
[rtbFile1]  [rtbFile2]   ← 左右のRichTextBoxに色分け表示
```

- `List<DiffLine>` が中心的なデータモデル
- 比較結果はMainFormの `_lastDiffResult` フィールドに保持され、レポート出力時に再利用される

## 5. 差分検出アルゴリズムの詳細

### LCS（Longest Common Subsequence）方式

1. **DPテーブル構築** — O(m×n) の動的計画法
   - lines1[i] == lines2[j] なら dp[i,j] = dp[i-1,j-1] + 1
   - 異なれば dp[i,j] = max(dp[i-1,j], dp[i,j-1])

2. **バックトラック** — テーブル末尾から先頭へ逆走査
   - 一致 → Equal
   - ファイル1にしかない → Deleted
   - ファイル2にしかない → Added
   - 結果をreverse()して正順にする

3. **Modified変換** — 後処理
   - 連続する [Deleted, Deleted, Added, Added] のようなパターンを検出
   - ペアになるものを Modified(変更前→変更後) に変換
   - ペアにならない余りはDeleted/Addedのまま残す

### 計算量
- 時間: O(m×n)　空間: O(m×n)
- m, n はそれぞれのファイルの行数
- 数千行のファイルまでが実用的な範囲

## 6. Commonライブラリ（DesktopKitシリーズ共通基盤）

FolderViewer と同一のCommonライブラリを共有している。

**BaseFormが提供する統一仕様:**
- タイトル形式: 「DesktopKit — {コンポーネント名}」
- ウィンドウサイズ: 初期800x600、最小640x480
- フォント: メイリオ 9pt
- ステータスバー: 画面下部に常時表示

**StatusHelperが提供するステータス表示:**
- ShowInfo: デフォルト色（黒）
- ShowSuccess: 緑色テキスト
- ShowError: 赤色テキスト

**FileDialogHelperが提供するダイアログ:**
- SelectFile: ファイル選択ダイアログ
- SaveFile: 保存ダイアログ（初期ディレクトリ・フィルタ・デフォルトファイル名の指定可）

## 7. 機能追加時の技術的な考慮点

| やりたいこと | 影響範囲 | 備考 |
|-------------|---------|------|
| インライン差分（文字単位ハイライト） | DiffEngine + DiffRenderer に追加 | RichTextBoxの文字単位の色制御が必要 |
| 出力形式の追加（HTML等） | DiffReporter に新メソッド追加 | 既存への影響小 |
| 空白無視・大文字小文字無視オプション | DiffEngine の比較部分 + MainFormにUI | 比較時のオプション化 |
| フォルダ比較 | 新クラス追加 + MainForm大幅変更 | 大規模な変更 |
| 拡張子制約の緩和/撤廃 | MainForm の拡張子チェック部分 | 小規模な変更 |
| エンコーディング手動選択 | MainForm にUI追加 + 読み込みロジック | 中規模の変更 |
| 性能改善（大ファイル対応） | DiffEngine のアルゴリズム変更 | Myers差分等への置き換えが候補 |
| クリップボードからの貼り付け比較 | MainForm にUI追加 | テキスト入力エリアの追加が必要 |
| 行番号クリックでジャンプ | DiffRenderer にイベント追加 | RichTextBoxのクリック位置から行特定 |
