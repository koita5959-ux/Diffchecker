# Diffchecker v1.0 — 現状機能仕様書

> この資料は、ソースコードを参照できない環境での機能面の打ち合わせ用に、
> Claude Code がソースコードを読み取って作成したものです（2026-03-29）。

---

## 1. アプリケーション概要

| 項目 | 内容 |
|------|------|
| 名称 | Diffchecker |
| シリーズ | 便利アプリシリーズ — DesktopKit |
| 用途 | 2つのテキスト系ファイルを比較し、差分を色分け表示・テキストレポートとして書き出す |
| 想定ユーザー | 設定ファイルの変更確認、コードレビュー、ドキュメントの版管理など |
| 対応OS | Windows 10 / Windows 11（64bit） |

---

## 2. 画面構成

アプリケーションは**1画面構成**で、上から順に以下のエリアで構成される。

```
┌──────────────────────────────────────────────────┐
│  タイトルバー: 「DesktopKit — Diffchecker」       │
├──────────────────────────────────────────────────┤
│  ファイル1: [ 選択中のパス（読取専用）    ] [参照] │
│  ファイル2: [ 選択中のパス（読取専用）    ] [参照] │
│  [比較]                                           │
├──────────────────────────────────────────────────┤
│  ファイル1              │ ファイル2               │
│ ─────────────────────── │ ──────────────────────  │
│    1: 共通行の内容      │    1: 共通行の内容      │
│    2: 変更前の内容 ████ │    2: 変更後の内容 ████ │
│    3: 削除された行 ████ │       ---          ████ │
│       ---          ████ │    3: 追加された行 ████ │
│    4: 共通行の内容      │    4: 共通行の内容      │
│                         │                        │
│      （左右分割、ドラッグで幅調整可能）            │
├──────────────────────────────────────────────────┤
│  [レポート出力]                                   │
├──────────────────────────────────────────────────┤
│  ステータスバー: 「比較完了 — 変更: 1件 / ...」   │
└──────────────────────────────────────────────────┘
```

- ウィンドウ初期サイズ: 800 x 600px
- 最小サイズ: 640 x 480px
- フォント: メイリオ 9pt
- ウィンドウは画面中央に表示
- 左右パネルはSplitContainerで50:50に分割（スプリッターをドラッグして調整可能）

## 3. 機能詳細

### 3-1. ファイル選択

- 「参照」ボタンをクリックすると、Windows標準の**ファイル選択ダイアログ**が開く
- 対応ファイル形式（フィルタに表示される拡張子）:
  - テキスト系: `.txt` `.csv` `.json` `.xml` `.html` `.htm` `.css` `.md` `.log`
  - プログラム: `.js` `.ts` `.jsx` `.tsx` `.cs` `.csproj` `.py` `.php` `.vbs` `.ps1` `.bat` `.cmd` `.sh` `.sql`
  - スタイル: `.scss` `.sass` `.less`
  - 設定: `.ini` `.yaml` `.yml` `.toml` `.env` `.properties` `.conf` `.cfg` `.reg`
  - 「すべてのファイル」も選択可能
- **ドラッグ&ドロップ対応**: エクスプローラーからファイルをテキストボックスにドロップして指定可能
- 両方のファイルが指定されると「比較」ボタンが有効になる

### 3-2. 拡張子チェック

- 「比較」ボタン押下時に、2つのファイルの拡張子が一致するか検証する
- **大文字・小文字は区別しない**（.TXT と .txt は同一扱い）
- 拡張子が異なる場合:
  - 警告メッセージボックスが表示される:「拡張子が異なるファイルは比較できません。同じ形式のファイルを選択してください。」
  - ステータスバーに赤文字で「エラー：拡張子が一致しません」と表示
  - 比較は**実行されない**

### 3-3. エンコーディング検出

- ファイル読み込み時に自動でエンコーディングを判定する
- 判定順序:
  1. **UTF-8 BOM付き** — 先頭3バイトが `0xEF 0xBB 0xBF` の場合
  2. **UTF-8（BOMなし）** — 厳密モードでデコードを試行し、成功した場合
  3. **Shift_JIS (コードページ932)** — UTF-8でデコードできなかった場合のフォールバック
- ユーザーがエンコーディングを手動指定する機能はない

### 3-4. 差分検出アルゴリズム

- **LCS（Longest Common Subsequence: 最長共通部分列）** ベースの行単位比較
- 比較は**文字列の完全一致**（1文字でも異なれば「異なる行」と判定）

**差分の4種類:**

| 種類 | 意味 | 色 |
|------|------|-----|
| Equal（一致） | 両ファイルで同一内容の行 | 色なし（白背景） |
| Modified（変更） | 両ファイルに存在するが内容が異なる行 | 黄色 `#FFFDE6` |
| Deleted（削除） | ファイル1にのみ存在する行 | 赤色 `#FFE6E6` |
| Added（追加） | ファイル2にのみ存在する行 | 緑色 `#E6FFE6` |

**Modified（変更）の検出方法:**
- LCSアルゴリズム本体はEqual / Deleted / Added の3種類のみを検出する
- 後処理として、**連続するDeleted行とAdded行の組**を検出し、対になるものをModifiedに変換する
- 例: Deleted(A) → Deleted(B) → Added(C) → Added(D) の場合 → Modified(A→C), Modified(B→D) に変換

### 3-5. 色分け表示

- 左パネルにファイル1、右パネルにファイル2の内容を表示する
- 各行の先頭に**行番号**（4桁右揃え + コロン + スペース）を付与
  - 例: `   1: ` `  42: ` ` 100: `

**行の対応関係と表示ルール:**

| 差分種類 | 左パネル（ファイル1） | 右パネル（ファイル2） |
|---------|---------------------|---------------------|
| Equal | 行番号 + 内容（白背景） | 行番号 + 内容（白背景） |
| Modified | 行番号 + 変更前の内容（黄色背景） | 行番号 + 変更後の内容（黄色背景） |
| Deleted | 行番号 + 削除された内容（赤背景） | `      ---`（赤背景）※プレースホルダ |
| Added | `      ---`（緑背景）※プレースホルダ | 行番号 + 追加された内容（緑背景） |

- プレースホルダ `---` により、左右の行が**常に同じ高さに揃う**
- 両パネルのRichTextBoxは**読み取り専用**（編集不可）
- **折り返し表示なし**（長い行は横スクロールで確認）

### 3-6. スクロール連動

- 左右のパネルは**垂直スクロールが連動**する
- 片方をスクロールすると、もう片方が同じ位置まで自動スクロールする
- Win32 API（GetScrollPos / SetScrollPos / SendMessage）を使用して実現

### 3-7. レポート出力（テキスト書き出し）

- 「レポート出力」ボタンは、比較実行後に有効になる（初期状態は無効）
- クリックすると、Windows標準の**名前を付けて保存ダイアログ**が開く
- デフォルトファイル名: `diff_{ファイル1名}_vs_{ファイル2名}_{yyyyMMdd}.txt`
- 出力エンコーディング: UTF-8（BOM付き）、改行コード: CRLF

**出力内容は2セクション構成:**

**セクション1 — 全行比較（マーク付き）:**
```
差分レポート
============================================================
比較元: C:\path\to\file1.txt
比較先: C:\path\to\file2.txt
作成日: 2026-03-29
============================================================

--- 全行比較 ---

     1:  共通行の内容
~    2:  変更前の内容  →  変更後の内容
-    3:  削除された行の内容
+    4:  追加された行の内容
     5:  共通行の内容
```

**マーク記号の意味:**
| マーク | 意味 |
|-------|------|
| （スペース） | Equal（一致） |
| `~` | Modified（変更）— 「変更前  →  変更後」の形式で1行に表示 |
| `-` | Deleted（削除）— ファイル1の行番号で表示 |
| `+` | Added（追加）— ファイル2の行番号で表示 |

**セクション2 — 差分サマリー:**
```
--- 差分サマリー ---

[変更] 行 2:
  元: 変更前の内容
  先: 変更後の内容

[削除] 行 3:
  元: 削除された行の内容

[追加] 行 4:
  先: 追加された行の内容

--- 集計 ---
変更: 1件 / 追加: 1件 / 削除: 1件 / 合計: 3件
============================================================
```

- 差分がない場合は「差分はありません。」と表示

### 3-8. 保存先の記憶

- 最後にレポートを保存したディレクトリを記憶し、次回の保存ダイアログで初期ディレクトリとして使用する
- 設定は `%AppData%\DesktopKit\Diffchecker.json` にJSON形式で保存
- 保存される情報は `Diffchecker.LastSavePath` のみ

### 3-9. ステータスバー

- 画面下部に常時表示
- 初期状態: 「準備完了」
- 比較完了（差分あり）: 「比較完了 — 変更: X件 / 追加: Y件 / 削除: Z件」（緑文字）
- 比較完了（差分なし）: 「比較完了 — 差分はありません」（緑文字）
- 拡張子エラー時: 「エラー：拡張子が一致しません」（赤文字）
- レポート保存完了: 「レポートを保存しました：{ファイルパス}」（緑文字）

## 4. エラー処理

| 状況 | 挙動 |
|------|------|
| 拡張子不一致 | メッセージボックス + ステータスバーに赤文字表示。比較は中断 |
| エンコーディング判定失敗 | Shift_JISにフォールバック（エラー表示なし） |
| 設定ファイルの読み書き失敗 | 静かに無視 |
| ファイル選択ダイアログでキャンセル | 何もしない |

## 5. 現状の制約・特記事項

| 項目 | 内容 |
|------|------|
| 比較単位 | 行単位のみ。文字単位・単語単位のハイライトはない |
| 比較方法 | 完全一致比較。空白の違い、大文字小文字の違いも「異なる」と判定 |
| 拡張子制約 | 異なる拡張子のファイル同士は比較できない |
| エンコーディング | UTF-8 と Shift_JIS の自動判定のみ。EUC-JP等は未対応 |
| 出力形式 | テキスト（.txt）のみ。HTML等のリッチな形式は非対応 |
| 性能 | LCSアルゴリズムは O(m×n) の計算量。数千行程度のファイルが実用範囲 |
| インライン差分 | 変更行（Modified）の中で、具体的にどの部分が変わったかの表示はない |
| 3ファイル比較 | 非対応。2ファイルのみ |
| フォルダ比較 | 非対応。ファイル単体の比較のみ |
| 比較結果の編集 | 不可（読み取り専用表示） |
| バイナリファイル | 対象外（テキスト系ファイルのみ） |
