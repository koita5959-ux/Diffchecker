## Phase 5：タブ指定カラム出力オプション

### 目的

レポート出力に「カラム表示モード」をオプションとして追加する。
ファイル1とファイル2の内容を左右に並べ、罫線文字で区切ることで、テキストエディタ上でもUIの左右分割に近い視認性を実現する。

**この機能の価値：**
現状のレポートは変更行を `元の内容  →  新しい内容` の1行形式で表記する。短い内容なら問題ないが、ファイル1とファイル2の差が大きい場合、テキストでは見づらくなる。左右カラム化＋罫線文字で、テキストファイルとしての可読性を高める。

**設計方針：**
- 既存のレポート出力（デフォルト）はそのまま維持する
- 「カラム表示モード」はオプションであり、押し付けない設計とする
- カラム表示ではスペースパディング＋罫線文字（`│`）で左右を揃える（タブ文字は使用しない。エディタ依存で崩れるリスクを回避するため）

### レポートフォーマット（カラム表示モード）

カラム表示モードがONの場合、セクション1「全行比較」のフォーマットを以下に変更する。
セクション2「差分サマリー」は変更なし（現状のまま）。

#### カラムレイアウトの計算

最大横幅をWとする（デフォルト120）。マージンMは4文字固定とする。

```
構成: [マーク1文字][空白1文字][行番号4桁][: ][内容（左カラム）][マージン][│][空白][行番号4桁][: ][内容（右カラム）]
       1          1         4      2       左カラム幅      M=4    1   1    4      2       右カラム幅
```

- 固定幅部分：マーク(1) + 空白(1) + 行番号(4) + コロンスペース(2) = 8文字（左側）
- マージン＋区切り部分：マージン(4) + 罫線(1) + 空白(1) = 6文字
- 右側行番号：行番号(4) + コロンスペース(2) = 6文字
- 内容幅（左右合計）= W - 8 - 6 - 6 = W - 20
- 左カラム内容幅 = (W - 20) / 2（端数切り捨て）
- 右カラム内容幅 = W - 20 - 左カラム内容幅

**マージンの役割：**
左カラムの内容と罫線 `│` の間に4文字分の空白を確保する。内容が罫線に密着すると視認性が悪くなるため、余白を設けて左右の区切りを明確にする。内容がカラム幅より短い場合は、マージン分も含めてスペースで埋める（結果として4文字以上の余白が空く）。内容がカラム幅いっぱいの場合でも、最低4文字の余白が保たれる。

**W=120の場合の計算例：**
- 内容幅合計：120 - 20 = 100文字
- 左カラム内容幅：50文字
- 右カラム内容幅：50文字
- フォールバック判定：左または右の内容が50文字を超える場合に `→` 形式に切り替え

**W=80（最小値）の場合の計算例：**
- 内容幅合計：80 - 20 = 60文字
- 左カラム内容幅：30文字
- 右カラム内容幅：30文字

#### ヘッダー行

```
差分レポート（カラム表示）
============================================================
比較元: C:\work\config_v1.txt
比較先: C:\work\config_v2.txt
作成日: 2026-03-29
最大横幅: 120文字
============================================================

--- 全行比較 ---

  ファイル1                                          │  ファイル2
─────────────────────────────────────────────────────┼──────────────────────────────────────────────────
```

- ヘッダーに「最大横幅: XXX文字」を追記する（読み手が表示崩れの原因を判断できるように）
- ファイル1/ファイル2のラベルを左右カラムの開始位置に合わせる
- 罫線は `─`（U+2500）を使用、交差部は `┼`（U+253C）を使用

#### 本体行の表示ルール

**Equal行（同一行）：**
```
   1:  host=localhost                                │   1:  host=localhost
```

**Modified行（変更行）：**
```
~  6:  log_level=info                                │   6:  log_level=debug
```
- 左にマーク `~`、左側にファイル1の行番号と内容、右側にファイル2の行番号と内容

**Deleted行（削除行）：**
```
-  9:  retry=3                                       │
```
- 左にマーク `-`、右側は行番号なし・内容なし（空白＋罫線のみ）

**Added行（追加行）：**
```
                                                     │ +11:  feature_flag=on
```
- 左側は空白、右側にマーク `+` ＋ファイル2の行番号と内容
- Added行のマーク `+` は右カラムの行番号の前に置く

#### 内容がカラム幅を超える場合

左カラムまたは右カラムの内容がカラム幅を超える場合は、その行のみ現状の `→` 形式にフォールバックする。

```
~  6:  very_long_line_that_exceeds_column_width=old_value  →  very_long_line_that_exceeds_column_width=new_value
```

- フォールバック行でも行番号はファイル1の行番号を使用する
- この判断は行ごとに行う。超えない行はカラム形式、超える行は `→` 形式が混在してよい

#### 全角文字の幅計算

- 全角文字は2文字分としてカウントする
- 半角文字は1文字分としてカウントする
- これにより日本語テキストのファイルでもカラム位置が揃う
- 判定基準：Unicode East Asian Width がWide (W) またはFullwidth (F) のものを全角とする
  - 実装が複雑になる場合は、char値が0x100以上を全角とする簡易判定でも可（ClaudeCodeの技術判断に委ねる）
